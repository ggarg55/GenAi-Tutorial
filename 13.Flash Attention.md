## 1. Why Attention Is Expensive

Transformers power most modern AI systems, but their self‑attention mechanism has a fundamental cost: **quadratic time and memory complexity** in the sequence length \(n\).

For a sequence of \(n\) tokens, attention computes an \(n \times n\) matrix of scores. When \(n\) grows large, this quickly becomes impractical:

| Sequence Length \(n\) | Number of Attention Scores (\(n^2\)) |
|-----------------------|--------------------------------------|
| 1,000                 | 1,000,000                            |
| 10,000                | 100,000,000                          |
| 100,000               | 10,000,000,000                       |

Storing and processing such large matrices limits the length of sequences we can handle and slows down training and inference.

## 2. GPU Memory: HBM vs. SRAM

Modern GPUs have two main types of memory, with very different characteristics:

| Memory Type | Role                | Speed (approx.) | Size (typical) |
|-------------|---------------------|-----------------|----------------|
| **HBM**     | Main storage        | 1–2 TB/s        | 40–80 GB       |
| **SRAM**    | On‑chip shared memory | 19 TB/s       | 20–50 MB       |

Think of HBM as a **library shelf**—it holds all your data, but fetching from it is relatively slow.  
SRAM is your **desk**—only a few items fit, but you can work on them instantly.

In standard attention, data is moved repeatedly between HBM and SRAM, wasting time on memory transfers. The goal of Flash Attention is to **minimize these transfers** by keeping as much computation as possible in SRAM.

---

## 3. Standard Attention: The Bottleneck

Given query \(Q\), key \(K\), and value \(V\) matrices (stored in HBM), the standard attention computation does:

1. Load \(Q\) and \(K\) from HBM → compute \(S = QK^\top\) → write \(S\) back to HBM.
2. Load \(S\) from HBM → compute \(P = \text{softmax}(S)\) → write \(P\) back to HBM.
3. Load \(P\) and \(V\) from HBM → compute \(O = PV\) → write \(O\) back to HBM.

Each step involves expensive reads and writes of the \(n \times n\) matrix \(S\) and \(P\). For long sequences, this memory traffic dominates runtime.

---

## 4. Tiling: The Core Idea

Instead of processing the whole matrix at once, we split the inputs into **blocks** (tiles) that fit entirely into SRAM. We then compute attention block‑by‑block, keeping only the necessary statistics to combine results later.

**Visualization:**

```
Full attention matrix (n × n):
┌─────────────────────────────────┐
│ ████████████████████████████████ │
│ ████████████████████████████████ │
│ ████████████████████████████████ │
│ ████████████████████████████████ │
└─────────────────────────────────┘

Split into blocks that fit in SRAM:
┌────────┬────────┬────────┬────────┐
│ Block  │ Block  │ Block  │ Block  │
│   1    │   2    │   3    │   4    │
├────────┼────────┼────────┼────────┤
│ Block  │ Block  │ Block  │ Block  │
│   5    │   6    │   7    │   8    │
├────────┼────────┼────────┼────────┤
│ Block  │ Block  │ Block  │ Block  │
│   9    │  10    │  11    │  12    │
├────────┼────────┼────────┼────────┤
│ Block  │ Block  │ Block  │ Block  │
│  13    │  14    │  15    │  16    │
└────────┴────────┴────────┴────────┘
```

We process each block, compute partial attention, and update running statistics. This way, we never need to hold the full \(n \times n\) matrix in memory.

## 5. Incremental Softmax: Making Blocks Work

Softmax normally requires the entire set of values to compute the denominator. How can we compute it block by block?

### Standard Softmax Formula

For a vector \(x\):

\[
\text{softmax}(x_i) = \frac{e^{x_i - m}}{\sum_j e^{x_j - m}}
\]

where \(m = \max(x)\) ensures numerical stability.

### Processing Two Blocks

Suppose we have two blocks of scores: \(x^{(1)}\) and \(x^{(2)}\). After processing the first block, we store:

- \(m_1 = \max(x^{(1)})\)
- \(l_1 = \sum_i e^{x^{(1)}_i - m_1}\)

When we see the second block, the new global maximum is \(m_2 = \max(m_1, \max(x^{(2)}))\).

To get the final softmax, we must **adjust** the first block’s contributions:

- Adjust old sum: \(l_1' = l_1 \cdot e^{m_1 - m_2}\)
- Compute new sum: \(l_2 = \sum_j e^{x^{(2)}_j - m_2}\)
- New total sum: \(l = l_1' + l_2\)

Now the final softmax values are:

- For elements in block 1: \(\frac{e^{x^{(1)}_i - m_1} \cdot e^{m_1 - m_2}}{l} = \frac{e^{x^{(1)}_i - m_2}}{l}\)
- For elements in block 2: \(\frac{e^{x^{(2)}_j - m_2}}{l}\)

This matches exactly what you would get by concatenating the blocks and computing softmax on the whole vector.

### Numerical Example

Take \(x = [2, 1, 0]\), processed as block 1 = \([2, 1]\) and block 2 = \([0]\).

- Block 1: \(m_1 = 2\), \(l_1 = e^{0} + e^{-1} = 1 + 0.3679 = 1.3679\)
- Block 2: \(m_2 = \max(2, 0) = 2\), \(l_2 = e^{0-2} = 0.1353\)
- Adjust old sum: \(1.3679 \times e^{2-2} = 1.3679\)
- Total \(l = 1.3679 + 0.1353 = 1.5032\)
- Final softmax:  
  \(\frac{1}{1.5032} = 0.665\), \(\frac{0.3679}{1.5032} = 0.245\), \(\frac{0.1353}{1.5032} = 0.090\)

This matches the full softmax of \([2,1,0]\).

## 6. The Flash Attention Algorithm

Combining tiling and incremental softmax, the Flash Attention algorithm (simplified) proceeds as follows:

### Inputs
- \(Q, K, V\) in HBM (size \(n \times d\))
- SRAM size \(M\)

### Steps

1. **Determine block sizes**  
   Let \(B_c = \lfloor \frac{M}{4d} \rfloor\) (number of key/value blocks)  
   Let \(B_r = \min(B_c, \lfloor \frac{M}{4d} \rfloor)\) (number of query blocks)

2. **Initialize**  
   Output \(O = 0\) (matrix of zeros, \(n \times d\))  
   Statistics: \(m = -\infty\) (vector of length \(n\)), \(l = 0\) (vector of length \(n\))

3. **Outer loop** over key/value blocks \(K_j, V_j\) (size \(B_c \times d\)):  
   - Load \(K_j, V_j\) from HBM to SRAM.

4. **Inner loop** over query blocks \(Q_i\) (size \(B_r \times d\)):  
   - Load \(Q_i\) from HBM to SRAM.  
   - Compute \(S_{ij} = Q_i K_j^\top\) (in SRAM).  
   - Find \(m_{\text{new}} = \max(m_i, \text{rowmax}(S_{ij}))\).  
   - Compute \(P_{ij} = e^{S_{ij} - m_{\text{new}}}\) (element‑wise).  
   - Update \(l_{\text{new}} = l_i \cdot e^{m_i - m_{\text{new}}} + \text{rowsum}(P_{ij})\).  
   - Update output:  
     \[
     O_i = O_i \cdot \frac{l_i \cdot e^{m_i - m_{\text{new}}}}{l_{\text{new}}} + \frac{P_{ij} V_j}{l_{\text{new}}}
     \]  
   - Update \(m_i = m_{\text{new}}\), \(l_i = l_{\text{new}}\).

5. **Return** \(O\).

All intermediate matrices (\(S_{ij}\), \(P_{ij}\)) are kept in SRAM. Only the final output \(O\) is written back to HBM.

## 7. A Concrete Example: Preserving Context

Does block‑wise processing lose long‑range dependencies? No—because each query eventually sees all keys through incremental updates.

Consider the sentence: **The cat is sitting on the sofa**

Focus on the word **sitting**. In standard attention, it looks at all words and assigns:

- High relevance to *cat* (who is sitting)
- High relevance to *sofa* (where it sits)
- Medium relevance to *on*
- Low relevance to articles

Now process with Flash Attention using two blocks:

**Block 1:** *The cat is sitting*  
- For the query *sitting*, we compute partial attention to *cat* (high), *is* (low), *the* (low).  
- Store partial statistics.

**Block 2:** *on the sofa*  
- Load the second block.  
- For *sitting*, compute relevance to *on* (medium) and *sofa* (high).  
- Update global softmax statistics (max, sum) and adjust previous contributions accordingly.  
- Combine with the first block’s partial output.

After processing both blocks, the final attention weights for *sitting* are exactly the same as if we had processed the whole sentence at once. **No context is lost.**

## 8. Benefits and Trade‑offs

| Aspect | Standard Attention | Flash Attention |
|--------|-------------------|-----------------|
| **Memory usage** | \(O(n^2)\) | \(O(n)\) |
| **Memory traffic** | High (many HBM writes) | Low (few HBM writes) |
| **Speed** | Slow | Up to 3× faster for GPT‑2 |
| **Exactness** | Exact | Exact (same result) |
| **Training speedup** | – | ~15% end‑to‑end wall‑clock time |

**Key insights:**

- Flash Attention is **not an approximation**—it produces identical outputs to standard attention.
- Speedups come from minimizing data movement between HBM and SRAM.
- It enables training with much longer sequences on the same hardware.

## 9. FlashAttention‑2 and Beyond

The original Flash Attention paper (2022) introduced tiling and recomputation. A follow‑up, **FlashAttention‑2** (2023), further improves parallelism and work partitioning, leading to even faster execution. Both are widely adopted in modern LLM training and inference frameworks.
